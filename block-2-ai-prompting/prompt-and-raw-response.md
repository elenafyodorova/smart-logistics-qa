# Промпт и необработанный (raw) ответ LLM

## Промпт

```text
Ты выступаешь в роли QA-инженера. Помоги составить чек-лист тест-кейсов
для формы создания заявки на перевозку груза в логистической платформе
"УЛ Лайт" (MVP).

Форма содержит следующие поля:
1. Город отправления — автокомплит/выпадающий список, данные подгружаются из БД
2. Город назначения — автокомплит/выпадающий список, данные подгружаются из БД
3. Вес груза — числовое поле, единица измерения кг, максимальное значение 20 000 кг
4. Объем груза — числовое поле, единица измерения м³, максимальное значение 82 м³
5. Дата погрузки — date picker, нельзя выбрать дату в прошлом
6. Кнопка "Рассчитать стоимость" — отправляет POST-запрос к API,
   в ответ возвращает стоимость в рублях либо ошибку, если маршрут
   между указанными городами не найден

Составь чек-лист тест-кейсов, охватывающий:
- позитивные сценарии (happy path)
- негативные сценарии (некорректный ввод, ошибки валидации)
- граничные значения (boundary values) для числовых полей и даты

Для каждого тест-кейса укажи: поле/функционал, тип проверки,
входные данные, ожидаемый результат.

Оформи результат в виде таблицы.
```
