# Чек-лист / тест-кейсы по ЭТрН

**Основание:** BPMN-диаграмма процесса ЭТрН и XSD-схемы из `https://www.nalog.gov.ru/`.

## 1. Основной бизнес-процесс ЭТрН

| ID    | Проверка                                                             | Ожидаемый результат                                                                    |
|-------|----------------------------------------------------------------------|----------------------------------------------------------------------------------------|
| BP-01 | Создать новую ЭТрН                                                   | Открывается новая карточка документа со статусом `Черновик`                            |
| BP-02 | Заполнить обязательные сведения грузоотправителя, перевозки и груза  | Все обязательные поля принимаются системой без ошибок                                  |
| BP-03 | Сохранить заполненную ЭТрН как черновик                              | Документ сохраняется, данные не теряются                                               |
| BP-04 | Попробовать отправить документ с незаполненными обязательными полями | Отправка блокируется, пользователь видит список ошибок валидации                       |
| BP-05 | Подписать и отправить корректно заполненную ЭТрН                     | Документ уходит следующему участнику процесса и меняет статус на `Отправлен`/`Передан` |
| BP-06 | Получение документа оператором ЭДО                                   | Документ маршрутизируется адресату без искажения реквизитов                            |
| BP-07 | Получение ЭТрН перевозчиком                                          | Перевозчик видит все переданные реквизиты и статус получения                           |
| BP-08 | Подтверждение получения ЭТрН перевозчиком                            | В системе фиксируется подтверждение, документ переводится на следующий этап            |
| BP-09 | Проверка перевозчиком сведений по грузу                              | При отсутствии замечаний документ продолжает движение по процессу                      |
| BP-10 | Фиксация замечаний перевозчиком                                      | Система сохраняет замечания и переводит документ в ветку корректировки                 |
| BP-11 | Повторная отправка/корректировка после замечаний                     | История изменений сохраняется, новые сведения становятся актуальными                   |
| BP-12 | Инициировать изменение грузополучателя и/или порта назначения        | Создаётся сценарий изменения, документ уходит на подтверждение по BPMN-ветке           |
| BP-13 | Подтверждение изменения перевозчиком                                 | Изменение либо принимается, либо отклоняется с сохранением причины                     |
| BP-14 | Передача данных о погрузке/выгрузке водителем                        | Данные фиксируются и доступны другим участникам процесса                               |
| BP-15 | Получение груза грузополучателем                                     | Грузополучатель видит актуальные данные по перевозке и грузу                           |
| BP-16 | Подтверждение приёма груза грузополучателем                          | Документ получает финальное подтверждение по грузу                                     |
| BP-17 | Фиксация замечаний грузополучателем                                  | Замечания сохраняются, процесс не закрывается как успешный                             |
| BP-18 | Завершение процесса после всех подтверждений                         | ЭТрН переходит в финальный статус и становится недоступной для обычного редактирования |
