# Негативные сценарии проверки ЭТрН

## Общая информация

Электронная транспортная накладная (ЭТрН) является юридически значимым документом и требует обязательного заполнения реквизитов, валидации данных и корректного обмена между участниками перевозки посредством ЭДО и электронной подписи. С 01.09.2026 использование ЭТрН становится обязательным для участников грузоперевозок.

---

# Негативный сценарий №1

## Потеря интернет-соединения при подписании и отправке ЭТрН перевозчиком

### Цель проверки

Убедиться, что система корректно обрабатывает отсутствие интернет-соединения при подписании и отправке ЭТрН и информирует пользователя о невозможности завершить операцию.

### Предусловия

* Пользователь авторизован в системе под ролью «Перевозчик».
* Заявка на перевозку заполнена корректно.
* ЭТрН готова к подписанию.

### Шаги

1. Открыть форму оформления ЭТрН.
2. Заполнить все обязательные поля.
3. Отключить интернет-соединение.
4. Нажать кнопку «Подписать и отправить».
5. Дождаться результата обработки операции.

### Ожидаемый результат

* Система отображает понятное сообщение об отсутствии интернет-соединения.
* Пользователь получает уведомление о том, что документ не был отправлен.
* Заполненные данные сохраняются в черновике.
* Статус ЭТрН не изменяется на «Отправлена».
* Пользователь может повторить отправку после восстановления соединения.

### Риски при отсутствии обработки

* Потеря введённых данных.
* Ошибочное представление пользователя об успешной отправке документа.
* Срыв сроков согласования перевозки.
* Нарушение бизнес-процесса оформления ЭТрН.

# Негативный сценарий №2

## Возможность сохранения ЭТрН с пустым значением поля «Дата доставки»

### Цель проверки

Проверить обязательность заполнения поля даты и корректность валидации документа перед сохранением или отправкой.

### Предусловия

* Пользователь создает новую ЭТрН.
* Поле «Дата доставки» является обязательным согласно бизнес-логике документа.

### Шаги

1. Создать новую ЭТрН.
2. Заполнить все обязательные поля кроме поля «Дата доставки».
3. Нажать кнопку «Сохранить» или «Отправить».
4. Проверить результат выполнения операции.

### Ожидаемый результат

* Система запрещает сохранение или отправку документа.
* Поле подсвечивается как обязательное.
* Отображается сообщение:

  > «Поле "Дата доставки" обязательно для заполнения».
* Документ не переводится на следующий этап бизнес-процесса.

### Риски при отсутствии обработки

* Некорректный расчет сроков перевозки.
* Нарушение логики маршрутизации груза.
* Ошибки при расчете транзитного времени.
* Нарушение требований к оформлению перевозочных документов.
